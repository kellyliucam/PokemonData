## Overview

For this project, I used Claude as my AI coding assistant to help me build an interactive web tool for looking up Japanese Pokémon names and for finding the specific index number of each one. The goal was to pull data from the Bulbapedia website and create a simple website to run the functions easily. I was originally planning on doing a python script, but I thought that for this kind of project a website would be more suitable and easier to navigate. Using a web interface (html) also made it more interactive and visually appealing.


## What I Learned 

Using AI in this project taught me a lot about how to work with coding assistants and the meaning of “vibe coding”. I learned that the most important part when dealing with AI is the prompt, being as specific as possible is necessary to achieve what you really want to create, since small changes in wording can easily change the results. I also realized the importance of debugging and fixing the code myself. Originally, the code the AI gave me had some issues with the interface and display, so I had to understand the problem and figure out how to fix it.

## The Process

The process was mostly straightforward. When I asked Claude to generate specific functions to scrape the Pokémon names and images, it actually found one additional data source online that I had not provided and even managed to link it with an existing repository in GitHub that had the source. This was unexpected because I only asked the AI to work with the website I specified. I tested the generated code and checked the output to figure out if the code needed any adjustments to ensure the data displayed correctly on the website. Honestly, without the help of AI, I probably would have taken way longer to build my idea from scratch and figuring out the code along would have been very time-consuming.

## Concerns and Confusion

There were times when the AI was confusing and kept misunderstanding my prompts about some issues I found with the code. Sometimes it misunderstood my prompts, especially when I was trying to fix small issues in the code or adjust the interface. I kept having to rephrase my instructions multiple times before it gave me back something useful, which was frustrating at times. Also, while the code generated technically worked it did not display what I originally wanted it to do so I had to start from scratch multiple times.
Another issue I had was that I initially tried building this project in Python. However, I kept running into errors and after many failed attempts and troubleshooting, I was not able to fix the issues. Because of this, I decided to switch to an HTML/JavaScript approach.
